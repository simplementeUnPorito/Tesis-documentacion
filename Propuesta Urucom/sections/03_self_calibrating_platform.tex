Fig.~\ref{fig:calibration_flow} summarizes, first, the sequence run by the digital hardware described next. At boot, after a gain change, or upon command, the firmware loads the stored DAC values, verifies the offsets, and restores the measurement path without a full run if all stages are within tolerance; otherwise it processes PGA, BP, SUM, and LP sequentially, settling after each multiplexer switch and FIR-history reset, bounded by a watchdog and per-stage sample limit. During acquisition the controller is inactive, the multiplexer returns to the LP output, and all VDAC codes remain constant: the method changes the DC operating point without introducing a time-varying feedback loop into the acquired signal.

The PSoC digital filter block is reused between the two operating modes (Fig.~\ref{fig:digital_path}): during acquisition it implements the geophone filter, and during calibration its coefficients switch to an aggressive low-pass response that provides a stable DC-level estimate for the stage being measured. Hardware reuse here reduces CPU intervention and preserves the existing acquisition architecture; acquisition coefficients and signal path are restored at the end of calibration. Successful DAC codes are stored in EEPROM, one record per PGA gain setting (hardware class, gain code, stage/valid-stage mask, four DAC codes, version, CRC); a failed run never overwrites a valid record.

\subsection{Control Law and Calibration Parameters}

For each stage, the filtered measurement $\hat{y}_i[n]$ produces the ADC-domain error $e_i[n] = r_i-\hat{y}_i[n]$, where $r_i=0$ for the current differential geophone configuration. The error is first mapped to the 8-bit DAC domain:
\begin{equation}
 \varepsilon_i[n] = \operatorname{round}\!\left(
 \frac{e_i[n]V_{\mathrm{ADC}}N_{\mathrm{DAC}}}
 {N_{\mathrm{ADC}}V_{\mathrm{DAC}}}\right),
 \label{eq:scale}
\end{equation}
with $V_{\mathrm{ADC}}=5000$~mV, $N_{\mathrm{ADC}}=262144$, $V_{\mathrm{DAC}}=4080$~mV, and $N_{\mathrm{DAC}}=255$ in the implemented configuration.

The calibration architecture only requires the selected stage be measurable and the trim DAC provide sufficient control range, so the controller can be chosen per application. In this prototype, a positional PI law was used for its simplicity:
\begin{equation}
 u_i^*[n]=u_{i,0}+s_i\left[
 \frac{k_{p,i}}{|K_i|}\varepsilon_i[n]+\frac{k_{i,i}}{|K_i|}
 \sum_{q=0}^{n}\varepsilon_i[q]\right],
 \label{eq:pi}
\end{equation}
where $u_{i,0}$ is the feedforward code, an initial DAC value near stage $i$'s expected operating point; $s_i\in\{-1,+1\}$ is the injection polarity; and $K_i>0$ is the magnitude of the DC gain from the DAC's injection point to the shared measurement node -- i.e., how much a static VDAC code change at stage $i$ moves the measured node, expressed in ADC-equivalent counts per VDAC code (Eq.~\ref{eq:scale}). $K_i$ is a nominal design value, fixed for BP, SUM, and LP ($2$, $7.89$, and $6$, reflecting downstream gain, not individually measured per board); for PGA, $K_i$ and $s_i$ are recomputed from the active gain code; its feedforward code $u_{i,0}$ was hand-set to the DAC value for gain $\times50$ -- the circuit's power-up gain -- and left fixed. For this deployment $k_{p,i}=0$ for all four stages (Table~\ref{tab:cal_params} lists only $k_{i,i}$): response speed is not a priority for a calibration that triggers once after deployment, so the general law reduces to a pure integrator.

The command $u_i^*[n]$ of Eq.~\ref{eq:pi} is saturated to the valid VDAC interval and limited to one code per sample, with conditional integration for clamping anti-windup. The deadband is $\Delta_i=\max(1,\lceil1.2|K_i|\rceil)$ DAC codes; convergence requires the error to stay in the same quantized bucket for a configurable number of samples, and a final adjacent-code trial is kept only if it reduces the filtered error. If a stage misses its deadband before a per-stage timeout, the DAC stays at the best code found, the stage is flagged not converged, and the sequence moves on, bounding worst-case time without leaving any stage arbitrary or saturated.

Table~\ref{tab:cal_params} lists the parameters used, to support reproduction. Settle is 512 samples (0.20~s) for PGA/BP/SUM and 1024 (0.39~s) for LP; lock requires 512 or 1024 consecutive in-deadband samples, and refinement settles a further 1024--2048 samples. Timeouts are worst-case ceilings, not the typical duration shown in Fig.~\ref{fig:calibration_trace}.

The 8-bit VDAC resolution suffices here; a finer step could use a PSoC IDAC/resistor or a buffered topology \cite{InfineonAN60305} -- an implementation choice, not a limitation, since controller, range, and step size can all adapt while keeping the same measure--estimate--correct sequence.
