At boot, after a gain change, or on command, the firmware loads stored DAC codes and verifies the offsets (Fig.~\ref{fig:calibration_flow}). Since DC estimation assumes a stationary geophone and sufficiently slow internal-state change over the lock window, an accepted operating point returns directly to acquisition; otherwise PGA, BP, SUM, and LP are processed sequentially, with finite-impulse-response (FIR) filter reset and settling after each switch. A watchdog and sample limits bound the sequence. During acquisition the controller is inactive, LP is selected, and the VDAC codes remain constant.

The shared digital filter implements the geophone response during acquisition and a 128-tap low-pass FIR DC estimator during calibration (Fig.~\ref{fig:digital_path}); its state and acquisition path are restored afterward. A gain-indexed electrically erasable programmable read-only memory (EEPROM) record stores the valid-stage mask, four DAC codes, version, and cyclic-redundancy check (CRC). A failed run may hold its bounded best codes for the current sequence but never overwrites a valid record.

\subsection{Control Law and Calibration Parameters}

For each stage, the ADC reports the node voltage relative to $V_{\mathrm{ref}}$. Thus, $\hat{y}_i[n]=0$ denotes a physical node at $V_{\mathrm{ref}}$, not at 0~V. The variables $\hat{y}_i[n]$, $r_i$, and $e_i[n]$ are signed ADC counts; the filtered differential measurement gives $e_i[n] = r_i-\hat{y}_i[n]$, with $r_i=0$. The error is first mapped to the 8-bit DAC domain:
\begin{equation}
 \begin{aligned}
 \varepsilon_i[n]
 &= \operatorname{round}\!\left(e_i[n]\frac{q_{\mathrm{ADC}}}{q_{\mathrm{DAC}}}\right)\\
 &= \operatorname{round}\!\left(
 \frac{e_i[n]V_{\mathrm{ADC,span}}N_{\mathrm{DAC}}}
 {N_{\mathrm{ADC}}V_{\mathrm{DAC,span}}}\right),
 \end{aligned}
 \label{eq:scale}
\end{equation}
where $q_x=V_{x,\mathrm{span}}/N_x$ is the step size and $N_x$ is the number of quantization levels, not the maximum code. Here $N_{\mathrm{ADC}}=2^{18}$ and $N_{\mathrm{DAC}}=2^8$. At nominal $V_{\mathrm{DDA}}=5$~V, $V_{\mathrm{ADC,span}}=5000$~mV and $q_{\mathrm{ADC}}=5000/2^{18}$~mV/count; physically, the differential span follows $V_{\mathrm{DDA}}$. The VDAC's ideal 256-bin span is 4096~mV, so $q_{\mathrm{DAC}}=16$~mV/code, while code 255 reaches 4080~mV \cite{InfineonPSoC,InfineonAN60305}.

The rounding in Eq.~\ref{eq:scale} gives an integer error in equivalent DAC codes. An overly narrow band can alternate indefinitely between adjacent codes when the ideal point is not realizable; the implementation therefore adopts the conservative one-step floor
\par\noindent
\begin{minipage}[c]{0.45\columnwidth}
\footnotesize
\begin{equation}
 \Delta_{i,\min}^{\mathrm{impl}}=\max(1,\lceil|K_i|\rceil),
 \label{eq:deadband}
\end{equation}
\end{minipage}%
\hfill
\begin{minipage}[c]{0.54\columnwidth}
\footnotesize
\begin{equation}
 \bar{\varepsilon}_i[n]=
 \begin{cases}
 0, & |\varepsilon_i[n]|\leq\Delta_i,\\
 \varepsilon_i[n], & \text{otherwise},
 \end{cases}
 \label{eq:deadband-error}
\end{equation}
\end{minipage}
\par
Inside the band, Eq.~\ref{eq:deadband-error} suppresses integration; lock and refinement still govern convergence. Table~\ref{tab:cal_params} lists adopted values satisfying $\Delta_i\geq\Delta_{i,\min}^{\mathrm{impl}}$.

Under the assumptions of Section~\ref{sec:platform-design}, the architecture requires a measurable stage output and sufficient trim authority and range. The controller is written in positional proportional-integral (PI) form,
\begin{equation}
 c_i^*[n]=c_{i,0}+s_i\left[
 \frac{k_{\mathrm{P},i}}{|K_i|}\bar{\varepsilon}_i[n]+\frac{k_{\mathrm{I},i}}{|K_i|}
 \sum_{q=0}^{n}\bar{\varepsilon}_i[q]\right],
 \label{eq:pi}
\end{equation}
Here $c_{i,0}$ is a feedforward code near the expected operating point and, under the adopted error convention, $s_i=\operatorname{sgn}(K_i)$. The four physical $K_i$ values in Table~\ref{tab:cal_params} are positive, so $s_i=+1$. The gain magnitudes follow from the topology and resistor ratios; the gain-2 two-op-amp input of \cite{CypressAN60319}, followed by the programmable-gain block (Fig.~\ref{fig:analog_path}), gives nominal differential signal gain $2\,\mathrm{Gain}_{\mathrm{PGA}}$, whereas its DC VDAC-to-node gain is $K_{\mathrm{PGA}}=1$. The reported runs used $\boldsymbol{c}_0=(156,156,156,156)$ for (PGA, BP, SUM, LP) and the $k_{\mathrm{P},i}$ and $k_{\mathrm{I},i}$ values in Table~\ref{tab:cal_params}.

The command is clamped and slew-limited to one code per sample, with conditional integration for anti-windup. Convergence requires a fixed sample count in one quantized bucket; an adjacent-code trial is retained only if it lowers the filtered error. At timeout, the best code is held, the stage is flagged, and the sequence advances. Codes are stored only if all controllable stages succeed; otherwise the prior EEPROM record remains.
