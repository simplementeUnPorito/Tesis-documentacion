## 2. Mandatory manuscript requirements

The manuscript must satisfy every item in this section.

### 2.1 Language and originality

- [ ] The complete manuscript is written in **English**.
- [ ] The work is original and has not been previously published.
- [ ] Reused material is properly cited, including material from the authors' own previous publications.
- [ ] The paper does not contain uncited copied or closely paraphrased material.
- [ ] The manuscript is consistent with the IEEE plagiarism policy.

### 2.2 Template and layout

- [ ] The manuscript uses the **IEEE Conference A4 double-column template**.
- [ ] The paper contains **no more than 5 pages**, including references.
- [ ] The initial and final submissions both comply with the same 5-page limit; no additional pages are available.
- [ ] Page size is **A4**, not US Letter.
- [ ] All margins match the official IEEE conference template.
- [ ] No text, equation, table, figure, caption, header, or footer invades the side, top, or bottom margins.
- [ ] The PDF layout remains valid after PDF/PDF-A processing.
- [ ] The manuscript contains no author photographs or biographies.

### 2.3 Authors and affiliations

- [ ] Every author is shown on the first page.
- [ ] Each author's full name, affiliation, and e-mail address are complete and correct.
- [ ] Author order in the PDF matches the author order registered in EDAS.
- [ ] Every author has or will create an EDAS account.
- [ ] All co-authors are included in the EDAS paper record.
- [ ] The corresponding/first-author information is internally consistent.

### 2.4 PDF requirements

- [ ] The deliverable is a **single PDF file**.
- [ ] The PDF opens correctly and contains all pages, figures, tables, equations, and references.
- [ ] The PDF contains **no embedded Internet hyperlinks**.
- [ ] No visible URL is implemented as a clickable link annotation.
- [ ] The PDF contains no unexpected attachments, comments, annotations, or form fields.
- [ ] Fonts and mathematical symbols render correctly.
- [ ] Figures remain legible at normal viewing size and in print.
- [ ] The final PDF has been checked independently from the LaTeX source.

> **Important:** URUCON states that embedded Internet links may trigger an alert during initial upload and cause rejection at the final-paper upload stage.

## 3. Figures, graphics, tables, and equations

- [ ] Every figure and table is cited in the text before or at its first appearance.
- [ ] Figures, pictures, drawings, tables, and captions remain inside the IEEE margins.
- [ ] All figure text, axis labels, legends, and annotations are readable.
- [ ] Figures are understandable in grayscale when color is not essential.
- [ ] Captions accurately describe what is shown and do not claim more than the figure demonstrates.
- [ ] Table values match the values discussed in the abstract, results, discussion, and conclusion.
- [ ] Units, symbols, decimal precision, signs, and significant figures are consistent.
- [ ] Every equation is numbered when referenced.
- [ ] Every symbol is defined at first use.
- [ ] Equation variables and parameter values are consistent with the implementation described in the text.

## 4. References and citation integrity

- [ ] Every in-text citation has a corresponding reference-list entry.
- [ ] Every reference-list entry is cited in the manuscript.
- [ ] Reference numbering follows order of first appearance.
- [ ] Bibliographic details, DOI values, years, titles, authors, and venues are correct.
- [ ] Each cited source actually supports the claim for which it is cited.
- [ ] No fabricated, unverifiable, or placeholder references remain.
- [ ] Reused material from the authors or co-authors is cited where required.
- [ ] Reference formatting is consistent with IEEE style.
- [ ] References fit within the overall 5-page limit.

## 5. Required technical-quality review

These checks are not presented by URUCON as formatting rules, but they are required for a coherent, review-ready IEEE paper.

### 5.1 Coherence

- [ ] The title, abstract, introduction, methods, results, discussion, and conclusion describe the same contribution.
- [ ] All numerical claims are consistent across text, equations, tables, captions, and figures.
- [ ] Hardware settings, gains, sample rates, resolutions, voltages, timing values, board counts, and run counts are consistent.
- [ ] The stated experimental scope matches the actual measurements.
- [ ] Claims about maximum, minimum, typical, representative, repeatable, stable, or converged behavior are supported by the reported evidence.
- [ ] The conclusion introduces no new result or experiment.
- [ ] Limitations stated in the discussion agree with the experimental scope.

### 5.2 Cohesion

- [ ] Each paragraph has a clear purpose and connects logically to the preceding and following paragraphs.
- [ ] Section transitions are explicit where needed.
- [ ] Terminology is consistent throughout the manuscript.
- [ ] Abbreviations are defined at first use and used consistently afterward.
- [ ] The paper avoids unnecessary repetition between the introduction, results, discussion, and conclusion.
- [ ] Pronouns and references such as “this method,” “this result,” and “the system” have unambiguous antecedents.

### 5.3 Completeness

- [ ] The abstract states the problem, proposed method, implementation context, validation scope, principal quantitative findings, and main limitation.
- [ ] The introduction identifies the gap and states the paper's contributions.
- [ ] The methods provide enough detail to understand and reproduce the architecture and calibration sequence.
- [ ] Experimental conditions are stated, including board count, gain setting, number of runs, measurement instrument, and compared conditions.
- [ ] Results distinguish measured facts from interpretation.
- [ ] Discussion explains significance, limitations, and generalizability without overstating the evidence.
- [ ] Conclusion answers the problem posed in the introduction.
- [ ] No placeholder text remains, including:
  - `TODO`
  - `TBD`
  - `XXX`
  - `pending`
  - `placeholder`
  - `falta medir`
  - temporary colors or editorial notes
- [ ] No section, figure, table, equation, citation, or cross-reference is missing.

### 5.4 Claim-to-evidence alignment

For each substantive claim, verify that it is supported by at least one of:

- a reported measurement;
- a table;
- a figure;
- an equation or derivation;
- a documented implementation detail;
- a verified external reference.

Report unsupported claims separately. Do not repair unsupported claims by inventing evidence.

- [ ] Final paper still complies with the 5-page limit.

- [ ] No author uploads more than two papers as submitting author.
- [ ] The applicable Author Registration fee is arranged if the paper is accepted.

## 8. Read-only review protocol

When this file is used for a read-only audit:

1. Do not modify `Urucom.tex`, bibliography files, figures, tables, scripts, or generated artifacts.
2. Inspect the complete source and the compiled PDF.
3. Report every item using one of these statuses:
   - `PASS`
   - `FAIL`
   - `PARTIAL`
   - `UNKNOWN`
   - `REQUIRES HUMAN CHECK`
   - `REQUIRES NEW DATA`
4. Prioritize findings as:
   - `CRITICAL`: submission blocker, contradiction, missing evidence, template/page/link failure, or unfinished content.
   - `HIGH`: issue likely to affect technical evaluation or reproducibility.
   - `MEDIUM`: clarity, cohesion, consistency, or presentation problem.
   - `LOW`: optional stylistic improvement.
5. For each finding report:
   - requirement;
   - file and exact location;
   - observed evidence;
   - why it matters;
   - proposed correction;
   - whether new data are required.
6. Do not classify an item as compliant when it cannot be verified.
7. Do not invent measurements, references, author metadata, or experimental details.
8. End with:
   - page count;
   - count of CRITICAL/HIGH/MEDIUM/LOW findings;
   - unresolved human checks;
   - unresolved new-data requirements;
   - overall status: `NOT READY`, `CONDITIONALLY READY`, or `READY FOR EDAS CHECK`.
